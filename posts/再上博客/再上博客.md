### 重建博客

接下来就是挑个好看点的blog，之前使用hexo部署，本地构建再推送过于麻烦（自己服务器上做钩子），而且换台电脑就要配个环境，还有一定的权限问题。

因此这里打算弄成和side_blog类似的，使用action来自动构建（仓库只存储posts）

最后选择了 [Hexo Aurora](https://aurora.tridiamond.tech/cn/) 这个主题。

:spotier[花里胡哨，但是还能接受]

因为旧的blog目录是

![1774451217740](1774451217740.png)

有辅助文件夹的，所以要现在hexo项目根目录下执行

```bash
npm install hexo-asset-link --save
```

下载上面这个插件并且修改下面的部分

![1774451283589](1774451283589.png)

:::caution

下图是修改过的，默认是false

:::

![1774451333451](1774451333451.png)

另外还有一点，hexo博客不要把图片等资源直接放在public目录下，而是放在source目录，public目录是通过hexo generate生成的，你的文件应该放在这个source目录下。

一般情况下，当要使用图片时，都是相对于网站url的根目录（如果以/开头）否则是相对当前页面。头像等资源可以直接放在source目录下的随便一个地方，实际使用时，不需要带上source目录

:::tip

例如,假设hexo根目录是blog，将图片放在blog/source/assets/avatar.ico
那么在link="/"的配置下，avatar="/assets/avatar.ico"

:::

hexo本身部署是要推送上来的，但是我这里由于是直接在服务器上运行，所以直接将

hexo server后台运行即可。

这里我使用豆包推荐的pm2（建议都在blog）

```bash
#安装
npm install -g pm2
#后台运行
hexo clean && hexo generate
pm2 start "hexo server" --name blog
#更改
hexo clean && hexo generate
pm2 restart blog
```

这样也方便配合action自动化

第二天发现面板登录不上去了，以为是网络问题，但是尝试了换端口，科学上网，依旧不行，最后发现是IP变了导致的

> 之前阿里云的安全组设置了放行所有流量，没有发现默认是和当前IP绑定的
>
> 此外，1panel如果在命令行中更改面板端口，安全组**不会自动放行**

:::tip

被阿里云墙表现为超时，如果要放行所有，访问来源写为0.0.0.0/0（添加授权对象->选择0.0.0.0/0->把原来的授权对象删掉）

:::

然后就是经典的卡图片加载的问题，上面下载完插件还是不够的，一开始我以为是生成时的图片路径问题，但是测试了多次后发现**只要刷新下页面图片就能显示出来**。

[终于解决了hexo开启懒加载后图片无法立即显示的问题_hexo lazyload-CSDN博客](https://blog.csdn.net/qq_41911142/article/details/124253167) 

直接卸载后，发现居然还是存在这个问题

偶然间我发现，重启完后，我停留在原来的文章页面，直接加载出来了图片，这很反常，尝试后发现：

从首页而来时

![1774510189190](1774510189190.png)

刷新后

![1774510226076](1774510226076.png)

这很奇怪，怎么会有这种不寻常事情

经过一番探索，发现这其实是个bug，这是因为我当时开了相对路径(`_config.yml`下的`relative_link`)，生成时的链接只有文件名称，加上/后恰好可以访问到。

关掉后图片的路径直接指向了根路径，兜兜转转回到起点。

理想下路径应该还有等同于文件名的目录

走了歪路，怎么办呢，**把上面的插件都删掉**，关闭相对路径，然后换一个插件：

```bash
npm install https://github.com/CodeFalling/hexo-asset-image --save
#如果之前有下载过这个插件，以及有类似功能的插件，请全部卸掉后安装
```

然后一切正常起来。最后放相关文件的配置（极个别选项不确定有没有影响）
